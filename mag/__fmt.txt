FileMaker es una plataforma que le permite construir aplicaciones que se adaptan exactamente a lo que necesitan las empresas. Las ventajas con relación a las aplicaciones prefabricadas es enorme: en las aplicaciones personalizadas la curva de aprendizaje es prácticamente inexistente. Estas aplicaciones pueden ir evolucionando en la medida que la empresa o sus empleados lo necesiten.

Como plataforma, Filemaker es única.Combina un motor de base de datos con herramientas de creación de interfaces. Esto nos permite crear, modificar o borrar la información y también crear informes y exportar los datos. Cientos de usuarios pueden acceder a la aplicación resultante de forma simultánea, desde un móvil o un navegador web. Filemaker es, sin dudas, una sofisticada arma de digitalización masiva.
Filemaker tiene más de 30 años en el mercado. Es un un producto de Claris, una empresa subsidiaria de Apple.

No existe ningún prerequisito para tomar este curso. En la comunidad de desarrolladores de Filemaker, hay profesionales de todo tipo - desde médicos hasta arquitectos-, pero también estudiantes y autodidactas. El único requisito es tener curiosidad, la necesidad de encontrar una solución a las deficiencias del sistema de información que use o sencillamente aprender a construir cualquier aplicación desde cero.

La plataforma tiene 2 componentes esenciales.
La aplicación Filemaker Pro que nos permite construir aplicaciones de base de datos en Filemaker.

Y Filemaker server: que es una aplicación de de servidor que nos permite compartir las aplicaciones construidas en Filemaker. Filemaker server nos permite acceder a la aplicación mediante cualquier navegador, la aplicación gratuita Filemaker Go o usando  Filemaker Pro en Windows o Mac. El conjunto de tecnologías que nos permite acceder via web, Claris lo denomina Webdirect. 
El interfaz es prácticamente idéntico al que vemos en la aplicación de escritorio. 
Este interfaz web se construye sin usar una sóla línea de código en HTML, CSS o Yávascript, ni necesita ningún otro lenguaje de servidor.

Una vez alojado en el servidor, tambien permite compartir  los datos con otros sistemas, mediante el protocolo estándar de bases de datos - llamado ODBC- o mediante su potente API de datos.
