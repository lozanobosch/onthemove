Una base de datos es simplemente un conjunto de datos estructurados con un contexto. Una agenda de teléfonos, un listado de libros, el conjunto de llamadas de nuestra factura de teléfonos o nuestro historial médico, son ejemplos claros de datos con estas características. 
Dicho de forma muy simple, una base de datos la constituyen tablas con nombres de columnas o campos para identificar el contenido y tantas filas o registros como información querramos almacenar. 
Es lo que miles de personas hacen cada día en excel. 
Y de la misma manera que en excel identificamos una celda con el tipo de valor que contiene - ya sea texto, número, fecha o cálculo- lo hacemos en los campos de nuestras tablas en la base de datos. 
Como todos sabemos la tipificación del campo es importante si queremos usarlo como referencia para un cálculo ya sea concatenar texto o calcular el descuento de una factura.
Pero hay una característica que hace que una base de datos sea diferente de una hoja electrónica: podemos relacionar las tablas con los datos. La razón de hacerlo es para evitar la redundancia de información. Por ejemplo, un curso escolar puede tener varios alumnos, un médico puede tener varias visitas, un número de teléfono puede tener varias llamadas o un cliente puede tener varias facturas. 
Filemaker es una base de datos relacional porque permite este tipo de vinculación entre sus tablas de datos. Hasta este punto no hay ninguna diferencia con otras bases de datos como Óracol, SQL Server o MySQL. la diferencia radica en las dos capas adicionales que tiene File Maker: una capa de presentación que permite construir el interfaz de la aplicación y una capa lógica, que no es más que los cálculos que usamos en los campos y en las rutinas o escrípts que se ejecutan con la intervención directa o indirecta de los usuarios.
